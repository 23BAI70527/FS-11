Experiment: React Routing Application (MST)

🎯 Aim

To develop a simple React application that demonstrates navigation between multiple pages using **React Router** and understand the concept of a **Single Page Application (SPA)**.

📌 Objectives

🔹 To understand the working of **React Router** in a React application.
🔹 To create multiple pages and navigate between them without refreshing the browser.
🔹 To learn how **Routes, Route, and Link components** are used for routing.
🔹 To organize a React project using a proper **folder structure**.
🔹 To implement a **404 Error Page** for invalid routes.

Tools / Technologies Used

1.React.js
2.Vite
3.React Router DOM
4.JavaScript (ES6)
5.HTML & CSS
6.Visual Studio Code
7.Node.js & npm

📖 About the Experiment

In this experiment, a React application named **MST** was created using **Vite**. The project demonstrates routing between two pages: **Home** and **About**. Navigation links are implemented using the **Link component** from React Router, allowing users to move between pages without reloading the browser.

The routing system is defined in the **App.jsx** file using **BrowserRouter, Routes, and Route** components. Each page is placed in a separate component inside the **pages folder**, which improves modularity and code organization.

A **404 Error Page** is also added to handle incorrect URLs so that users receive a clear message when a page is not found.

⚙️ Functionalities

1.Navigation between **Home Page** and **About Page**.
2,Single Page Application behavior (no full page reload).
3.Navigation bar with links** to switch between pages.
4.404 Error Page for invalid routes.
5.Organized **folder structure** for better project management.

 🎓 Learning Outcomes

1.📚 Understanding the concept of **client-side routing** in React.
2.⚛️ Learning how to use **React Router components** such as BrowserRouter, Routes, Route, and Link.
3.🧩 Gaining knowledge of **component-based architecture** in React.
4.📂 Understanding the importance of **modular code and proper folder structure**.
5.🛡️ Learning how to implement **error handling using a 404 page** in web applications.
